Enlaces avanzados
Etiquetas <a></a>
Atributos comunes Básicos, internacionalización, eventos y foco
Atributos propios

name = "texto": Permite nombrar al enlace para que se pueda acceder desde otros enlaces. Este atributo está cayendo en desuso, ya que cumple la misma función que el atributo id = "texto".

href = "url": Indica la URL del recurso que se quiere enlazar.

hreflang = "codigo_idioma": Idioma del recurso del enlazado. Puede ser "en", "es", "fr", etc. Ver códigos de idioma.

type = "tipo_de_contenido": Permite "avisar" al navegador sobre el tipo de contenido que se enlaza (imágenes, archivos, etc.) para que pueda preparase en caso de que no entienda ese contenido.

rel = "tipo_de_relacion": Describe la relación del documento actual con el recurso enlazado. Ver tipos de relación.

rev = "tipo_de_relacion": Describe la relación del recurso enlazado con el documento actual. Ver tipos de relación.

charset = "tipo_de_charset": Describe la codificación del recurso enlazado. Ver tipos de charset.

Descripción Se emplea para enlazar todo tipo de recursos

Enlaces a correos electrónicos

También hay enlaces que abren automáticamente el programa de correo electrónico del ordenador del usuario y establece la dirección de envío al valor indicado después de mailto: La sintaxis es la misma que la de un enlace normal, salvo que se cambia el prefijo http:// por mailto::

<a href="mailto:nombre@direccion.com" title="Dirección de email para solicitar más información">Enviar correo electrónico</a>

La sintaxis mailto: permite utilizarlo para otros ejemplos más complejos:

<a href="mailto:nombre@direccion.com,otro_nombre@direccion.com">Enviar correos electrónico</a>

<a href="mailto:nombre@direccion.com?subject=Solicitud de más información">Enviar correo electrónico con asunto</a>

<a href="mailto:nombre@direccion.com?body=Estaría interesado en solicitar más información sobre sus productos">Enviar correo electrónico con contenido</a>

Evidentemente, todas las opciones anteriores se pueden combinar entre sí para realizar ejemplos más avanzados.

La etiqueta <script> (tanto cuando enlaza, como cuando incluye directamente códigoede aparecer en cualquier parte del documento HTML, aunque normalmente se incluye dentro de la cabecera de la página (<head></head>).

Además, aunque la etiqueta <script> permite enlazcódigovarios lenguajes de programación, el uso habitual de <script> consiste en enlazar un archivo JavaScript externo:

<html>

  <head>

    <title>Página web con un enlace a un script</title>

    <script type="text/javascript" src="http://www.ejemplo.com/js/inicializar.js"></script>

  </head>

  <body>

    ...

  </body>

</html>

Links
Etiqueta <link></link>
Atributos comunes Básicos, internacionalización y eventos
Atributos propios

media = "tipo_de_medio": Indica el medio para el que debe aplicarse la relación.

href = "url": Indica la URL del recurso que se quiere enlazar. Los medios disponibles también están estandarizados, siendo los más comunes screen para los contenidos mostrados en pantalla, print para las impresoras y handheld para los dispositivos móviles.

hreflang = "codigo_idioma": Idioma del recurso del enlazado. Puede ser "en", "es", "fr", etc. Ver códigos de idioma.

type = "tipo_de_contenido": Permite "avisar" al navegador sobre el tipo de contenido que se enlaza (imágenes, archivos, etc.) para que pueda preparase en caso de que no entienda ese contenido.

rel = "tipo_de_relacion": Describe la relación del documento actual con el recurso enlazado. Ver tipos de relación.

rev = "tipo_de_relacion": Describe la relación del recurso enlazado con el documento actual. Ver tipos de relación.

charset = "tipo_de_charset": Describe la codificación del recurso enlazado. Ver tipos de charset.
Tipo de elemento En línea
Descripción Realza la importancia del texto que encierra (por defecto es interpretado visualmente en cursiva por los navegadores)

Ejemplos:

Al contrario que <script> la etiqueta <link> solamente se puede incluir dentro de la cabecera del documento. Se pueden añadir tantas etiquetas <link> como sean necesarias, pero siempre dentro de (<head></head>).

El uso habitual de la etiqueta <link> es el de enlazar las hojas de estilos CSS, favicon y RSS:

<html>

  <head>

    <title>Encabezado con enlaces a hojas de estilos, favicon y RSS</title>

    <link rel="stylesheet" type="text/css" href="/css/impresora.css" media="print" />

    <link rel="stylesheet" type="text/css" href="/css/movil.css" media="handheld" />

    <link rel="shortcut icon" href="/favicon.ico" type="image/ico" />

    <link rel="alternate" type="application/rss+xml" title="Resumen de todos los artículos del blog" href="/feed.xml" />

  </head>

  <body>

    ...

  </body>

</html>

También se puede utilizar para indicar que existe una versión de la página en otro idioma:

<html>

  <head>

    <title>English tutorial</title>

    <link lang="es" xml:lang="es" title="El tutorial en español" type="text/html" rel="alternate" hreflang="es" href="http://www.ejemplo.com/tutorial/espanol.html" />

  </head>

  <body>

    ...

  </body>

</html>

O para indicar que existe una versión de la página preparada para imprimir:

<html>

  <head>

    <title>Link para indicar versión para imprimir</title>

    <link media="print" title="El tutorial en PDF" type="application/pdf" rel="alternate" href="http://www.ejemplo.com/tutorial/documento.pdf" />

  </head>

  <body>

    ...

  </body>

</html>

O bien para indicar que existe una página que es índice de la página actual:

<html>

  <head>

    <title>Tutorial – Capítulo 5</title>

    <link rel="start" title="El índice del tutorial" type="text/html" href="http://www.ejemplo.com/tutorial/indice.html" />

  </head>

  <body>

    ...

  </body>

</html>
